Local repository

exo

  • Créer un repo R1

  • Valoriser le user local à user1

  • Indexer (stage) un fichier file1.txt

  • Commiter le fichier file1.txt

  • Modifier et indexer le fichier file1.txt

  • Cloner le repo R1 en un repo R2

  • Vérifier que dans R2 file1.txt ne contient pas la modification

  • Constater l’unique commit dans le repo R2

solution

$ git init R1
$ cd R1
R1 (master) $ git config user.name user1
R1 (master) $ date > file1.txt
R1 (master) $ cat file1.txt
lun. 27 févr. 2017 17:27:29

R1 (master) $ git add .
R1 (master) $ git commit -m"add file1"
R1 (master) $ date > file1.txt
R1 (master) $ cat file1.txt
lun. 27 févr. 2017 17:28:31

R1 (master) $ git add .
R1 (master) $ cd ..

$ git clone ./R1/ R2
$ cd R2
R2 (master) $ cat file1.txt
lun. 27 févr. 2017 17:27:29

R2 (master) $ git log

repo distant

exo

  • Créer un repo remote: git init repo-remote --bare

  • Cloner ce repo-remote dans un repo local nommé repo-local

  • Ajouter et commiter un fichier file1.txt dans repo-remote

  • Créer une branche develop dans repo-local

  • modifier et commiter file1.txt dans develop

  • "pusher" cette nouvelle branche dans repo-remote

  • visualiser la nouvelle dans repo-remote

solution

$ git init repo-remote --bare
$ git clone ./repo-remote repo-local
$ cd repo-local/
repo-local (master)  $ date > file.txt
repo-local (master)  $ git add .
repo-local (master)  $ git commit -m "file1.txt"
repo-local (master)  $ git branch develop
repo-local (master)  $ git checkout develop
repo-local (develop) $ date > file1.txt
repo-local (develop) $ git add .
repo-local (develop) $ git commit -m "update file1.txt"
repo-local (develop) $ git push origin develop
$ cd ../repo-remote
repo-remote (BARE:master) $ git branch -a -vv
