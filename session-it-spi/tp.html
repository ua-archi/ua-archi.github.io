Local repository

exo

  • Créer un repo R1

  • Valoriser le user local à user1

  • Indexer et commiter un fichier file1.txt

  • Modifier, indexer et commiter (amender le dernier commit) le fichier file1.txt

  • Modifier et indexer le fichier file1.txt

  • Cloner le repo R1 en un repo R2

  • Vérifier dans R2 que file1.txt ne contienne pas la modification

  • Constater l’unique commit dans le repo R2

solution (1/2)

$ git init R1
$ cd R1
R1 (master) $ git config user.name user1
R1 (master) $ echo "file1" > file1.txt
R1 (master) $ git add .
R1 (master) $ git commit -m"add file1"
R1 (master) $ echo "file1_amend" > file1.txt
R1 (master) $ git add .
R1 (master) $ git commit --amend -m"add file 1 amend"
R1 (master) $ echo "update file1" > file1.txt
R1 (master) $ git add .
R1 (master) $ cd ..
$ git clone ./R1/ R2

solution (2/2)

$ cd R2
R2 (master) $ cat file1.txt
 file1_amend
R2 (master) $ git log --oneline
 6614d6d add file 1 amend

repo distant

exo 1

  • Créer un repo "bare": git init repo-shared --bare

  • Cloner ce repo-shared dans un repo local nommé RL1

  • Ajouter et commiter un fichier file1.txt dans RL1

  • Créer une branche develop dans RL1

  • Modifier et commiter file1.txt dans develop

  • "Pusher" master et develop dans repo-shared

  • Visualiser la nouvelle branche dans repo-shared

solution exo 1

$ git init repo-shared --bare
$ git clone ./repo-shared RL1
$ cd RL1/
RL1 (master)  $ date > file1.txt
RL1 (master)  $ git add .
RL1 (master)  $ git commit -m "file1.txt"
RL1 (master)  $ git branch develop
RL1 (master)  $ git checkout develop
RL1 (develop) $ date > file1.txt
RL1 (develop) $ git add .
RL1 (develop) $ git commit -m "update file1.txt"
RL1 (develop) $ git push origin master
RL1 (develop) $ git push origin develop
$ cd ../repo-shared
repo-shared (BARE:master) $ git branch -a -vv

exo 2

  • Initialiser un repo local nommé RL2

  • Ajouter repo-shared comme nouveau remote (alias origin)

  • Synchroniser les références (miroirs) des branches distantes

  • Visualiser toutes les branches

  • Ajouter et commiter un fichier file2.txt dans develop

  • "Pusher" develop dans repo-shared

  • Synchroniser les branches locales de RL1

  • Visualiser le nouveau commit dans RL1

solution exo 2

$ git init RL2
$ cd RL2
RL2 (master) $ git remote add origin ../repo-shared/
RL2 (master) $ git fetch origin
RL2 (master) $ git branch -a -vv
RL2 (master) $ git checkout develop
RL2 (develop) $ date > file2.txt
RL2 (develop) $ git add .
RL2 (develop) $ git commit -m"add file2"
RL2 (develop) $ git push origin develop
$ cd ../RL1
RL1 (develop) $ git fetch origin
RL1 (develop) $ git rebase origin/develop
RL1 (develop) $ git log
